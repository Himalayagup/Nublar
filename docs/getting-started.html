Getting Started with Nublar

Installation

Nublar is a pure Python framework with no external dependencies. To get started:

1. Clone the Repository

git clone https://github.com/Himalayagup/Nublar.git
cd nublar

2. Create a Virtual Environment (Recommended)

# On Windows
python -m venv venv
venv\Scripts\activate

# On macOS/Linux
python3 -m venv venv
source venv/bin/activate
Note: While Nublar has no external dependencies, it's recommended to use a virtual environment to keep your project isolated.

Creating Your First App

Nublar provides a command-line tool to create new applications:

python mediator.py startapp myapp

Basic Configuration

Create a settings.py file in your project root:

# settings.py

# Debug mode
DEBUG_CODE = True

# Server settings
HOST = "127.0.0.1"
PORT = 4000

# Static files
STATIC_FOLDER = "static"

# Template settings
TEMPLATE_FOLDER = "templates"

# Database settings
DB_NAME = "db.sqlite3"

# Installed apps
ALL_APPS = [
    "myapp",
]

Creating Your First View

Open myapp/pages.py and add your first view:

from nublar.path import path
from response.http_response import text_response

@path("/hello")
def hello(request, method):
    return text_response("Hello, Nublar!")

Running the Server

Start the server:

python mediator.py start

You can also specify custom host and port:

python mediator.py start --host=0.0.0.0 --port=8000
Note: The server will be available at http://localhost:4000 by default. Visit http://localhost:4000/hello to see your first view in action.
