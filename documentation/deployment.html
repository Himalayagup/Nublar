Deployment

This guide covers everything you need to know about deploying your Nublar application to production. We'll cover different deployment options, configuration, and best practices.

Preparing for Deployment

Before deploying your application, there are several important steps to take:

1. Update Settings

# settings.py

# Production settings
DEBUG = False
SECRET_KEY = "your-secure-secret-key"  # Change this!
ALLOWED_HOSTS = ["yourdomain.com", "www.yourdomain.com"]

# Database settings
DB_NAME = "db.sqlite3"  # Consider using a more robust database in production

# Static files
STATIC_URL = "/static/"
STATIC_ROOT = "staticfiles/"  # Directory to collect static files
Important: Never use the default secret key in production. Generate a secure random key and keep it secret.

2. Collect Static Files

# Collect all static files into STATIC_ROOT
python mediator.py collectstatic

3. Database Migration

# Apply all migrations
python mediator.py migrate

Deployment Options

1. Traditional VPS (Virtual Private Server)

Deploying on a VPS gives you full control over your server environment.

Requirements

  • Linux server (Ubuntu recommended)
  • Python 3.8 or higher
  • Nginx or Apache web server
  • Gunicorn or uWSGI as WSGI server

Installation Steps

# Update system
sudo apt update
sudo apt upgrade

# Install Python and required packages
sudo apt install python3 python3-pip python3-venv nginx

# Create and activate virtual environment
python3 -m venv venv
source venv/bin/activate

# Install project dependencies
pip install -r requirements.txt
pip install gunicorn

# Install and configure Nginx
sudo apt install nginx
sudo systemctl start nginx
sudo systemctl enable nginx

Nginx Configuration

# /etc/nginx/sites-available/nublar
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location /static/ {
        alias /path/to/your/staticfiles/;
    }

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

Running with Gunicorn

# Start Gunicorn
gunicorn --workers 3 --bind 127.0.0.1:8000 run:app

# Using systemd service
# /etc/systemd/system/nublar.service
[Unit]
Description=Nublar Gunicorn Service
After=network.target

[Service]
User=youruser
Group=yourgroup
WorkingDirectory=/path/to/your/project
Environment="PATH=/path/to/your/venv/bin"
ExecStart=/path/to/your/venv/bin/gunicorn --workers 3 --bind 127.0.0.1:8000 run:app

[Install]
WantedBy=multi-user.target

2. Platform as a Service (PaaS)

PaaS providers offer easier deployment with less server management.

Heroku

# Procfile
web: gunicorn run:app

# runtime.txt
python-3.9.0

# Install Heroku CLI and deploy
heroku create your-app-name
git push heroku main
heroku run python mediator.py migrate

PythonAnywhere

# WSGI configuration file
import sys
path = '/home/yourusername/yourproject'
if path not in sys.path:
    sys.path.append(path)

from run import app as application

3. Docker Deployment

Docker provides consistent environments across different platforms.

Dockerfile

# Dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

RUN python mediator.py collectstatic --noinput
RUN python mediator.py migrate

EXPOSE 8000

CMD ["gunicorn", "--workers", "3", "--bind", "0.0.0.0:8000", "run:app"]

Docker Compose

# docker-compose.yml
version: '3'

services:
  web:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - .:/app
    environment:
      - DEBUG=False
      - SECRET_KEY=your-secret-key
    depends_on:
      - db

  db:
    image: postgres:13
    volumes:
      - postgres_data:/var/lib/postgresql/data/
    environment:
      - POSTGRES_DB=nublar
      - POSTGRES_USER=nublar
      - POSTGRES_PASSWORD=your-password

volumes:
  postgres_data:

Security Considerations

1. Environment Variables

Use environment variables for sensitive information:

# settings.py
import os

SECRET_KEY = os.environ.get('SECRET_KEY')
DEBUG = os.environ.get('DEBUG', 'False') == 'True'
ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', '').split(',')

2. SSL/TLS

Always use HTTPS in production. You can use Let's Encrypt for free SSL certificates:

# Install Certbot
sudo apt install certbot python3-certbot-nginx

# Obtain certificate
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

3. Security Headers

Add security headers to your Nginx configuration:

# Nginx configuration
add_header X-Frame-Options "SAMEORIGIN";
add_header X-XSS-Protection "1; mode=block";
add_header X-Content-Type-Options "nosniff";
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains";

Monitoring and Maintenance

1. Logging

Configure logging in your application:

# settings.py
LOGGING = {
    'version': 1,
    'disable_existing_loggers': False,
    'handlers': {
        'file': {
            'level': 'INFO',
            'class': 'logging.FileHandler',
            'filename': 'nublar.log',
        },
    },
    'loggers': {
        'nublar': {
            'handlers': ['file'],
            'level': 'INFO',
            'propagate': True,
        },
    },
}

2. Backup Strategy

  • Regular database backups
  • Static file backups
  • Configuration backups
  • Test restore procedures

3. Performance Monitoring

  • Monitor server resources (CPU, memory, disk)
  • Set up application performance monitoring
  • Configure error tracking
  • Set up uptime monitoring
Note: Regular maintenance and monitoring are crucial for keeping your application running smoothly in production.
